STRIDE Analysis
Potential Threats and Examples
Type of Threat Examples
Spoofing
  • An attacker creates a fake login page that looks identical to the real one, and sends emails to users asking them to log in. When users enter their credentials, the attacker captures them and uses them to log in to the real website as the victim.
  • Man-in-the-middle attacks, where the attacker intercepts communication between the user and the website.
  • Brute force attacks, where the attacker tries to guess the user's password by trying many different combinations.
Tampering
  • An attacker exploits a vulnerability in the website's code to insert malicious scripts that modify the data stored in the website's database (SQL injection attacks, where the attacker executes malicious code in the database,Cross-site scripting (XSS) attacks, where the attacker injects malicious code into the website) .
  • An attacker gains access to the website's administrative panel and uses it to alter or delete customer records or orders.
  • An attacker intercepts data being transmitted between the website and the user, and modifies it in transit to change the content displayed to the user.
Tampering
  • An attacker tampers with the website's configuration files or settings to change the behavior of the website.
Repudiation
  • An attacker deletes log files or alters their content to cover their tracks or to make it difficult to determine the extent of their actions.
  • An attacker tampers with the website's auditing system to make it appear that certain actions did not take place.
Information Disclosure
  • An attacker exploits a vulnerability in the website's code to gain access to sensitive information stored in the database(Data breaches).
  • An attacker intercepts data being transmitted between the website and the user, and captures sensitive information such as login credentials or financial details.(Eavesdropping)
  • An attacker gains access to the website's administrative panel and views or exports sensitive information such as customer records or order details.
  • An attacker convinces an authorized user to reveal their login credentials or to grant the attacker access to sensitive information.
Denial of Service
  • An attacker floods the website with traffic from multiple sources, overwhelming the website's servers and making it unavailable to its intended users. (DDoS)
  • An attacker targets the website's servers or infrastructure with attacks such as DB floods, disrupting the website's ability to function.
  • An attacker targets the website's domain name system (DNS) servers, preventing users from being able to resolve the website's address and access it.
  • An attacker targets the website's hosting provider or network infrastructure, disrupting the website's ability to function.
Elevation of Privilege
  • An attacker gains access to the website's administrative panel and uses it to grant themselves higher privileges or access rights.
